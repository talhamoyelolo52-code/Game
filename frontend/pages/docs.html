📚 Documentation

Everything you need to build Minecraft plugins with Kodari AI.

Quickstart

1
Open the IDE — click "Open IDE" in the top right or go to /generator.
2
Describe your plugin — for example: "Create a sword that shoots fireballs on right-click with a 5-second cooldown".
3
AI generates the code — wait ~30 seconds. You'll see the file tree populate.
4
Edit & refine — click any file to edit. Ask the AI chatbot to make changes: "Add a permission node for this command".
5
Build & download — click the ⚡ Build button. You'll get a .zip with the full project (Maven-built .jar included after compiling).

How it works

Kodari uses Google's Gemini 2.0 Flash model under the hood. The backend sends your prompt along with a system prompt that tells the AI to:

  • Write Java 17 code using the Bukkit/Spigot/Paper API
  • Return a structured JSON of files (Java source, plugin.yml, pom.xml)
  • Use modern Paper API patterns where possible
  • Include proper event listeners, commands, and config handling

The IDE then renders the files in Monaco (the same editor that powers VS Code) and lets you edit them in place.

Setup (self-hosted)

To run your own instance locally:

# Clone and install
cd kodari-clone/backend
npm install

# Add your Gemini API key
echo "GEMINI_API_KEY=AIza..." > .env

# Start
node server.js

# Open http://localhost:3000
💡 Get a free Gemini API key at aistudio.google.com/apikey. The free tier gives you plenty of generations per minute.

Writing good prompts

The more specific your prompt, the better the code. Here are some examples:

❌ Vague

Make a cool plugin

✅ Specific

Create a custom item (golden hoe named "Reaper's Scythe") that on right-click:
- Heals the user 2 hearts
- Applies Speed II for 5 seconds
- Has a 10-second cooldown shown in the action bar
- Plays entity.wither.spawn sound
- Permission: kodari.scythe.use
- Cooldown bypass for kodari.scythe.bypass

Great prompts include:

  • The feature — what should it do?
  • Triggers — when does it activate? (right-click, command, event, etc.)
  • Constraints — cooldowns, permissions, costs, limits
  • Visual/audio feedback — particles, sounds, messages
  • Edge cases — what if the player is in creative? PvP? Has no permission?

Building & exporting

When you click ⚡ Build, Kodari simulates a Maven build (since real Java compilation requires JDK on the server). The output is a .zip file containing:

MyPlugin.zip
├── src/main/java/com/kodari/MyPlugin.java
├── src/main/resources/plugin.yml
├── pom.xml
├── README.md
└── BUILD.txt  ← instructions to compile locally

To compile locally:

unzip MyPlugin.zip
cd MyPlugin
mvn clean package
# .jar will be in target/

API Reference

You can also call Kodari's backend directly:

POST /api/generate

{
  "prompt": "Create a lightning sword",
  "pluginName": "LightningSword",
  "mcVersion": "1.20.4"
}

POST /api/chat

{
  "message": "How do I add a cooldown?",
  "history": []
}

POST /api/export

Returns a ZIP file with all the generated files.

GET /api/templates

Returns the list of starter templates.

FAQ

Is this free?

Yes. You only pay for the Gemini API usage (which has a generous free tier).

Does it actually compile to .jar?

The server-side build is a simulation. To produce a real .jar you need to compile locally with Maven. Most plugin developers do this anyway since they want to test on their server.

Which MC versions work?

Any version from 1.18+. The AI is instructed to use modern Paper API patterns which work across 1.18-1.21.

Can I use this for production servers?

Absolutely. The generated code uses real Bukkit/Paper APIs and includes proper plugin.yml, permissions, and event registration. Just review it before deploying (as you would any AI-generated code).

Troubleshooting

"Gemini API key not configured"

Make sure your .env file in the backend folder has a valid GEMINI_API_KEY.

"Generation takes forever"

First request can take 10-20s due to Gemini cold start. Subsequent ones are faster.

"AI generated broken code"

Click in the chat: "The code has a syntax error on line X, please fix". The AI will regenerate.
